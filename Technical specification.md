### 1. ОБЩЕЕ ОПИСАНИЕ И НАЗНАЧЕНИЕ
Создание высокопроизводительного, отказоустойчивого скрипта на языке Python для зеркальной синхронизации директорий (аналог `robocopy SOURCE DEST /MIR /MT:12 /R:3 /W:5`). Скрипт ориентирован на выкачивание данных из сетевых папок (SMB) на локальные накопители Windows с сохранением прав доступа (ACL). 

Код должен быть оформлен в виде **одного монолитного файла**, работающего исключительно через изолированные функции, готового к будущему импорту в качестве модуля.

Файл readme.md - описывает для чего скрипт, как его использовать
файл history.md - при каждой правке, даже самой простой дописывает историю изменений.  

---

### 2. ПРАВИЛА ОРГАНИЗАЦИИ КОДА И СЛУЖЕБНЫХ ПУТЕЙ
1.  **Блок констант:** Располагается на **самой первой странице скрипта** (сразу после импорта модулей) для оперативного ручного редактирования параметров.
2.  **Дефолтные пути:** В блоке констант железно прописываются тестовые директории:
    ```python
    SOURCE_PATH = r"\\192.168.88.3\Отдел продаж"
    DEST_PATH = r"D:\BackUp\Отдел продаж"
    THREADS_COUNT = 16
    MAX_RETRIES = 3
    RETRY_DELAY = 5
    ```
3.  **Локализация служебных файлов:** Файлы `sync_main.log`, `sync_errors.log` и `sync_history.json` создаются и обновляются **строго в папке, где физически лежит сам скрипт** (вычисляется через `os.path.dirname(os.path.abspath(__file__))`).

---

### 3. КРИТИЧЕСКИЙ ФУНКЦИОНАЛ: СУПЕР-АВАРИЙНЫЙ ВЫХОД (BREAK ENGINE)
Во время разработки, отладки или работы скрипта администратор должен иметь возможность мгновенно прервать процесс без зависания консоли, порчи структуры файлов или удержания дескрипторов ОС.
* **Механизм:** Скрипт должен перехватывать комбинацию `Ctrl+C` (через `KeyboardInterrupt` или модуль `signal`) И нажатие горячей клавиши `F10` (через неблокирующий опрос клавиатуры `msvcrt.kbhit()`).
* **Поведение:** При срабатывании триггера, управляющий поток мгновенно выставляет глобальный флаг завершения (`threading.Event`). Потоки-воркеры прерывают текущие сетевые операции блочного копирования, закрывают открытые файловые дескрипторы, лог-движок сбрасывает буферы на диск, и скрипт мгновенно завершает работу с выводом сообщения: `[AVALANCHE EXIT]: Программа экстренно остановлена пользователем.` Требование обязательно для всех режимов UI.

---

### 4. АЛГОРИТМ ПРИНУДИТЕЛЬНОЙ СИНХРОНИЗАЦИИ ACL (ЭКВИВАЛЕНТ /SECFIX)
Скрипт работает по логике жесткого контроля безопасности:
* Если файл в `Source` изменился по размеру или дате — он копируется, и на него накатываются новые права доступа.
* **Режим /SECFIX (Всегда активен):** Если размер и дата модификации файла в источнике и цели полностью совпадают, скрипт **обязан** проверить бинарный дескриптор безопасности (`Win32 ACL`) через `pywin32`. Если права доступа на сетевой шаре изменились, воркер не копирует тело файла, а принудительно перезаписывает дескриптор безопасности на локальном файле, инкрементирует счетчик `acl_applied`, и заносит событие в журнал со статусом `[ACL_ONLY]`.

---

### 5. СИСТЕМА ЛОГИРОВАНИЯ (DUAL LOGGING)
Запись ведется в два файла в каталоге скрипта. В `main.log` пропущенные файлы **не пишутся**.
1.  **`sync_main.log`** (Только факты изменений):
    `ГГГГ-ММ-ДД ЧЧ:ММ:СС | СТАТУС | ВРЕМЯ_ОПЕРАЦИИ | РАЗМЕР | ОТНОСИТЕЛЬНЫЙ_ПУТЬ_ОТ_КОРНЯ_ШАРЫ`
    *Статусы:* `[COPY_OK]`, `[ACL_ONLY]`, `[DELETE_OK]`. Вместо ID потока пишется чистое время операции (например, `1.24s`).
2.  **`sync_errors.log`** (Все сбои и локи):
    `ГГГГ-ММ-ДД ЧЧ:ММ:СС | [CRIT_ERR] | Код ошибки Win32 / Описание | ОТНОСИТЕЛЬНЫЙ_ПУТЬ`

---

### 6. СТРУКТУРА ВОЗВРАТА ДАННЫХ И ИНТЕГРАЦИЯ
Вся логика инкапсулирована внутри функции `run_smart_sync()`. При вызове из внешних систем она возвращает Python-словарь (`dict`) со всей телеметрией: тайминги, объем, количество скопированных, удаленных и измененных (ACL) файлов, пиковые утилизации CPU/RAM.

# 📑 ПРИЛОЖЕНИЕ: ВЫДЕЛЕННЫЙ СПРАВОЧНИК ПО TUI И ЦВЕТОВОЙ ПАЛИТРЕ

Интерфейс утилиты жестко завязан на моноширинную сетку шириной **ровно 90 символов**. Отрисовка выполняется раз в 150–250 мс. Все цветовые маркеры реализуются через стандартные ANSI-последовательности.

### 🎨 1. Сводная таблица цветовых кодов (ANSI Спецификация)
* **Основной текст / Сетки:** Светло-серый (`\033[37m` / `#CCCCCC`)
* **Линии-разделители, IDLE-статусы:** Темно-серый (`\033[90m` / `#555555`)
* **Успех, 100%, Символ █:** Ярко-зеленый (`\033[92m` / `#00FF00`)
* **Пути, Максимумы скоростей:** Бирюзовый (`\033[96m` / `#00FFFF`)
* **Логи, Временные метки:** Желтый (`\033[93m` / `#FFFF00`)
* **Минимумы скоростей:** Ярко-красный (`\033[91m` / `#FF3333`)
* **Информационный блок (Первый запуск):** Розовый/Маджента (`\033[95m` / `#FF80FF`)
* **Аудит ACL (Фокус внимания):** Насыщенный оранжевый (256-bit ANSI: `\033[38;5;208m` / `#FF8000`)
* **Инверт-статус `[COPY]`:** Текст черный на бирюзовом (`\033[106;30m`)
* **Инверт-статус `[RTRY]`:** Текст черный на оранжевом (`\033[48;5;208;30m`)

### 💻 2. Конфигурация: РАБОЧИЙ UI (Active Sync Mode)

#### Критические правила разметки строк потоков:
1.  **Относительные пути источника (Source-ориентированность):** В строках потоков отображаются пути к файлам **источника**, а не локальной копии. При этом корень `\\192.168.88.3\Отдел продаж\` принудительно отсекается. Поток показывает только внутреннюю структуру папок.
2.  **Сжатие путей нового формата:** Если путь длиннее выделенной зоны, он сжимается строго по центру с разделителем из двух многоточий и пробела: `... ...`. 
    * *Пример:* `2026\Договоры\... ...\План_Продаж.xlsx`
3.  **Независимые спиннеры:** Анимационный символ (`/`, `-`, `\`, `|`) для каждого потока вычисляется изолированно. Шаг анимации меняется только тогда, когда конкретный поток считывает очередной чанк данных текущего файла. Простаивающий поток выводит статичную серую точку `•`.
4.  **Логика Первого Запуска:** Если файл истории сессии отсутствует, общий прогресс-бар `[███░░░]` полностью скрывается из интерфейса. На его место выводится центрированное розовое сообщение.
